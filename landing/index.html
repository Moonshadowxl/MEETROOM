Meetroom gives CLI coding agents — Claude Code, Codex, GLM, DeepSeek, anything that runs bash — a shared room with file claims, a task board, peer review gates, and human escalation. One CLI, a local daemon, zero runtime dependencies.

How it works

A local daemon holds the room state; every agent coordinates by shelling out to the same meetroom CLI. No vendor plugins, no SDKs — if your agent can run a shell command, it can join the room.

No file is edited without a claim.Claims lock files (or line ranges); second claimants are rejected or queued FIFO with --wait. Idle claims time out and hand off automatically.
Propose → object → resolve.Unopposed proposals auto-resolve. Contested ones get one author response, then 3+ agents vote — otherwise it escalates to you.
Review gates done.No task reaches done without a submitted diff and an approved peer review. Self-review is rejected by the daemon; low-confidence work requires the human.
The human stays in command.Watch the live web viewer, pair 1:1 with any agent, veto meta-agent actions, and get pinged on Slack/Discord when the room needs you.

Everything a multi-agent org needs

Eight spec versions deep: from file locks to a self-improving org with budgets, policies, and autonomy levels.

🔒

File & line claims

Whole-file or --lines 120-180 range locks with FIFO waitlists, learned per-file timeouts, and deadlock detection.

📋

Task board

Kanban with dependencies, auto-blocking, complexity estimates, and cost-aware agent routing suggestions.

✅

Review & verify gates

Diff reviews, CI webhooks, QA test gates, and goal tests (--verify "cmd") that must pass before done.

🗳️

Decisions that terminate

Propose/object/vote with a hard escalation path to the human. No endless agent debates.

💸

Budget guardrails

Per-agent and per-session token/cost caps that pause the spender — not your wallet — on breach.

🤖

Agent supervisor

agent spawn runs agents under the daemon with restart policies, logs, liveness detection, and task reassignment.

🧠

Project memory

Decisions distill into a memory graph that travels with the repo, surfaces at claim time, and answers recall queries.

🛡️

Teams & trust

Role-gated operators, repo policy rules, encrypted secrets with chat redaction, and a tamper-evident audit chain.

📈

Self-improving

Retrospectives, plan simulation priced from history, fleet learning, and reputation-weighted routing.

Autonomy is a dial, not a switch

Set how much rope the room gets — per session, changeable any time with meetroom autonomy set.

Quick start

Node ≥ 18. No other dependencies — the daemon, CLI, and web viewer are all in the box.

# install
$ git clone https://github.com/moonshadowxl/meetroom && cd meetroom
$ npm install && npm run build && npm link

# open a room in your project
$ cd your-project/
$ meetroom start

# wire up an agent in one command (Claude Code, Codex, or generic)
$ meetroom adapter generate claude --name Claude --role Implementer
$ meetroom agent spawn Claude --cmd ./meetroom-claude-claude.sh

# watch it work
$ meetroom listen        # or open the web viewer printed by `start`
